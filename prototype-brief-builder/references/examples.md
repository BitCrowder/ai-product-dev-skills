# Prototype Brief Examples

## Invocation

```text
Use $prototype-brief-builder to create a prototype brief for a mobile onboarding flow where users connect their calendar and get AI schedule suggestions.
```

## Output Skeleton

```markdown
# Prototype Brief: AI Calendar Onboarding

## Prototype Purpose
- Purpose: Usability test
- Fidelity: Medium
- Primary question: Can users understand why calendar access is needed?

## Primary Flow
1. User opens onboarding welcome screen.
2. User reviews value proposition.
3. User taps Connect Calendar.
4. System requests permission.
5. User sees first AI schedule suggestion.

## State Requirements
| Screen | Default | Error | Success |
|---|---|---|---|
| Calendar permission | Explain value and data use | Permission denied recovery | Connected confirmation |
```
